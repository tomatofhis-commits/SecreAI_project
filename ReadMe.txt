　SecreAI - ユーザーガイド (Ver 1.3.0)

この度は「AI秘書システム SecreAI」をご利用いただきありがとうございます。
本システムは、Googleの最新AI「Gemini」をメインエンジンとして使用し、会話・視覚（画面キャプチャ）・ネット検索を組み合わせた高度な応答を生成します。
ゲーム画面へのオーバーレイ表示と音声読み上げにより、あなたの頼れる秘書や相棒へと成長していくAIアシスタントです。

■公式リポジトリ（最新情報・詳細ドキュメント）
https://github.com/tomatofhis-commits/SecreAI_project

1. 同梱内容の確認
本プロジェクトおよびインストールフォルダは以下の主要なディレクトリとファイルで構成されています。

【1. ハブ・オーバーレイ (C# WPF)】
・secreAI.exe          : WPFで構築された統合操作パネル（ハブUI）です。
・RTtranslator_CS_Overlay.exe : 翻訳字幕およびAI応答テキストを透過描画するオーバーレイUIです。
・WPF/                 : フロントエンド（WPF C#）のソースコード一式です。
  - SecreAI_Hub_Window.cs : ハブUIの中核画面ロジック。
  - SecreAI_Hub_Overlay.cs : 縦幅100%・動的フォントスケール対応の応答オーバーレイ。
  - SecreAI_Hub_Server.cs : Python側と通信するための Flask 代替 HTTP サーバ。

【2. AIコアエンジン (Python scripts/)】
・scripts/              : AI対話や記憶、検索最適化を行うプログラム群です。
  - game_ai.py          : 音声認識/画像認識/会話を司るメインエンジン（スレッドプール自動制御）。
  - intersecting_ai.py  : Google Grounding と Tavily 検索を並列で行うハイブリッド検索AI。
  - update_memory.py    : 会話の要点を抽出し ChromaDB へ保存するバックグラウンド記憶定着タスク。
  - memory_viewer.py    : 長期記憶の検索・削除および統計ダッシュボード画面。
  - chromadb_pool.py    : ChromaDB接続をプールし、記憶のベクトル検索を高速化するモジュール。
  - api_cache_system.py : 同一の質問の再計算を防ぎ、応答速度を上げAPI費を節約するキャッシュ機構。

【3. リアルタイム翻訳エンジン (RTtranslator/)】
・RTtranslator_core.exe : 画面OCRおよび翻訳エンジン本体（PythonよりNuitkaビルド）。
・RTtranslator/        : リアルタイム翻訳機能のソースコード一式です。
  - capture.py          : ゲーム画面の超低遅延・高速画面キャプチャ。
  - ocr.py              : Windows内蔵OCR / PaddleOCRによる高精度文字検出。
  - translator.py       : Ollamaや翻訳APIを用いた多言語翻訳。

【4. アセット・設定・ビルドスクリプト】
・data/                 : 設定ファイル（config.json, rtt_config.json）、言語データ、対話履歴、一時音声wavが格納されます。
・memory_db/            : ChromaDB（ベクトルデータベース）による長期記憶の実データです。
・models/               : 翻訳エンジンの言語判定用AIモデルデータです。
・build.bat            : ハブのコンパイルからインストーラー作成までを自動実行するビルドスクリプト。
・build_wpf.bat        : MSBuildを呼び出し、C#フロントエンドをリリースビルドするスクリプト。
・build_python_runtime.py : 配布用ポータブルPythonを自動ダウンロード・最適構築するスクリプト。
・kill_zombies.bat     : バックグラウンドに残存したRTTコアやHubプロセスを一括強制終了するツール。

2. 事前準備
本システムを最大限活用するために、以下の準備を推奨します。
・Gemini APIキー (必須)
　会話の核心となるAIエンジンです。Google AI Studio から取得してください。
・Tavily APIキー (推奨)
　リアルタイムのウェブ検索を行うために必要です。tavily.com から取得してください。
・OpenAI APIキー (任意)
　gpt-5やgpt-5.4-miniなどの最新モデルを使用したい場合に必要です。platform.openai.com から取得してください。
・VOICEVOX (任意)
　日本語の高品質な読み上げを行う場合に必要です。別途インストールが必要です。
・Ollama (任意)
　ローカルAIによる高度な記憶整理・RT翻訳エンジンを利用する場合に必要です。
・SubtitLocar (任意)
　ローカルAI字幕システム連携を利用する場合に必要です。
　https://github.com/tomatofhis-commits/SubtitLocar


各APIの取得先:
・Gemini API: https://ai.google.dev/aistudio?hl=ja
・Tavily API: https://tavily.com/
・OpenAI API: https://platform.openai.com/

② Gemini APIキーの入力
③ Tavily APIキーの入力
④ VOICEVOXの連携設定
⑤ ローカルAI（Ollama）の使用確認

3. 基本的な使い方
ハブの各機能は以下の通りです。
・チャット入力: テキストボックスに書き込みエンターで送信。
・ボイスモード: マイクで直接AIと会話します。
・ビジョンモード: 現在選択されているウィンドウの画面をAIに見せながら質問できます。
・ターゲットウィンドウ: ビジョンモードでキャプチャする対象を選択します。
・おしゃべり停止: AIの読み上げや処理を即座に中断します。
・Good/Badフィードバック: AIの回答を評価します。AIはこれを分析して学習します。
・今日の前置条件: AIにその日の状況（遊んでいるゲームなど）を事前に伝えておけます。

4. Ver 1.3.0 の注目機能
◆ ローカルLLM（Ollama / LM Studio）のマルチプロバイダ対応
　これまでの Ollama に加え、OpenAI互換APIを持つ「LM Studio」との連携に完全対応しました。設定画面でラジオボタンを切り替えるだけで、それぞれの接続URL（Ollama / LM Studio）を保持したまま簡単に切り替えて利用できます。LM Studio のデフォルト接続先はポート 11435 に最適化されています。
◆ Google Grounding (Google Search) の月ごと制限カウントへの移行
　Grounding 検索の利用回数リセット周期を月単位に変更し、設定画面から今月の正確な累積回数を直感的に把握できるようになりました。
◆ 設定画面における Gemini 2.5 系列などの終了予定モデルを完全に排除
　公式に廃止が予定されている古いモデル（gemini-2.5等）の表示や警告ラベルを整理し、Grounding 固定モデルを「gemini-3.1-flash-lite-preview」に更新して最新環境への移行を完了しました。

5. 以前のバージョン（Ver 1.1.0）の注目機能
◆ RTトランスレーター統合
　メニュー「RTトランスレーター」からワンクリックでリアルタイム翻訳を開始・停止できます。
　OCRで画面上のテキストを読み取り、Ollamaの翻訳モデルにより各言語へリアルタイム翻訳し、ゲーム画面上に直接オーバーレイ表示します。

【パフォーマンス設定について】
設定画面の「RTトランスレーター」タブから、PCの負荷に合わせた調整が可能です。
・CPU使用制限: 翻訳処理に使用するCPUスレッド数をパーセント（25%〜100%）で制限できます。
・処理反応感度: 画面の変化をどれくらい敏感に検知するか調整します。「最大」にすると僅かな変化でも翻訳を更新し、「最低」にすると動きの激しいシーンでのCPU負荷を抑えられます。
・GPU/VRAM設定: 使用するGPUやビデオメモリの割り当てを指定し、スムーズな翻訳を実現します。

Stream Deckからは http://localhost:5001/api/translate にリクエストすることで翻訳の切り替えも可能です。

◆ ゲートキーパーAI
　ウェブ検索が必要な質問かどうかをAIが事前に判断し、不要な検索をカットします。
　これにより、回答速度が向上し、APIキーの消費も抑えられます。

◆ パフォーマンスダッシュボード
　「記憶管理」メニューから、AIモデル別（Gemini, OpenAI, Ollama等）の使用統計をグラフで確認できます。どのモデルがどれくらい活躍しているか一目で分かります。

◆ 高度なメモリ管理
　会話の内容は「短期・中期・長期」の3段階で記憶されます。
　長期記憶への保存時に不要な検索ログなどを自動でクリーンアップする機能が追加され、記憶の質が向上しました。

6. パフォーマンス最適化（自動実行）
以下の機能はシステムによって自動的に管理されます。
・APIキャッシュ: 同じ質問に対しては過去の回答を再利用し、爆速で返答します。
・検索キャッシュ: ネット検索結果を一時保存し、再検索の手間とコストを省きます。
・並列処理: 記憶の整理などをバックグラウンドで行うため、会話が途切れません。

■ インストーラー制作時の構成クリーンルール
本システムのインストーラー（setup_script.iss）をビルドする際は、開発者の個人データや不要な一時ファイルを配布物に含めないため、以下の除外ルールが Inno Setup 構成側で適用されています。
1. 設定ファイル（config.json, rtt_config.json）の除外
   - 既存の設定ファイルやAPIキーはインストーラーに含めず、ユーザーが各々初期設定を行えるようにします。
2. データベース・履歴ファイル（memory_db/*, chat_history.json, feedback_memory.json）の除外
   - 開発時の会話履歴や学習データ、および ChromaDB データベースフォルダは完全に除外してクリーンな状態で配布します。
3. キャッシュデータ（data/api_cache/*, data/search_cache/*, data/wav/*, translation_cache.json）の除外
   - 過去のAPIキャッシュ、Web検索キャッシュ、および生成された一時的な音声ファイル（.wav）は含めません。
4. ログファイル（*.log）および一時イメージファイルの除外
   - デバッグログやキャプチャ中の一時画像（temp_ss.png）等はすべて除外されます。
