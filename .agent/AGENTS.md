# AI開発行動ガイドライン (Pythonランタイム・起動パラメータの保護)

今後のすべての開発タスクにおいて、以下のルールを厳格に遵守してください。

## 1. ポータブルPythonランタイムおよび起動パラメータの変更禁止
- WPF Hub (`SecreAI_Hub`) から Python スクリプトを起動する際のコマンドライン引数（特に隔離用引数 `-I` など）や、環境変数のクリア（`ConfigureProcessEnvironment` 等によるPATHやシステム環境の全削除）を、ユーザーの明示的な事前承認なしに変更しないでください。
- 同梱しているポータブルPythonランタイム (`python_runtime`) の Python バージョンや、基本構成パッケージの構成、DLL of 抽出・ビルド手順を勝手に変更しないでください。

## 2. システム環境およびDLLの依存関係の保護
- 画面キャプチャ (dxcam, bitblt) や PaddleOCR など、グラフィックボードや DirectX、CUDA などのハードウェア支援機能を利用するモジュールは、OSのシステムDLL（`System32` 等）に依存しています。
- 環境変数を隔離することで、これらのシステムDLLへの参照パスが失われ、キャプチャエラーやGPUロードエラー（真っ黒な画面やモデル起動失敗）を招くため、システム環境と完全隔離するような設計変更を行わないでください。

## 3. ビルドバッチファイルの無断改変禁止
- `build_python_runtime.py` や `build_rtt_core.bat` などの構築スクリプトは、1.2.2以降の元の安定構成を基本として動作させるため、ビルドチェーンや引数の追加を勝手に行わないでください。

## 4. 修正手続きおよびビルド報告ルール
- バグなどの修正をするときは必ず修正計画案を提示し、許可を得てから行ってください。
- ユーザーから指示のない修正（自主的な追加・調整など）を行う場合は、ビルドを実行する前に、必ず「何を行ったのか」をユーザーに詳細に報告してください。
