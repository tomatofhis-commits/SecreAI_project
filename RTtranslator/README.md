# Real-Time Translator

ゲームやアプリケーションの画面をリアルタイムに翻訳し、透明なオーバーレイで直接画面上に表示する完全ローカル動作の翻訳ツールです。

[English README (英語版のREADMEはこちら)](README_en.md)

## 主な機能

- **進化したハイブリッドOCRエンジン**: 3つのプランから選択可能。
    - **Plan 1 (WinRT Only)**: 2パス処理により、高速性を維持しつつ精度を向上。
    - **Plan 2 (Hybrid)**: WinRTで枠を決め、PaddleOCRで精読・行結合を行う標準モード。
    - **Plan 4 (Dual-Scout Hybrid)**: 両エンジンで索敵し、PaddleOCRで仕上げる最高精度モード。
- **完全ローカル翻訳 (LLM)**: [Ollama](https://ollama.com/) を使用し、ローカル環境でLLM (`translategemma:4b` 等) を動かして翻訳。APIキーや外部サーバー通信は一切不要です。
- **新アルゴリズムによるUI最適化**: 翻訳後のテキストが元の枠に収まるよう、バイナリサーチ的に「枠内最大フォントサイズ」を自動調整します。視認性が大幅に向上しました。
- **インテリジェント・キャッシュ管理**: 起動時にキャッシュをリセットし、翻訳再開時にゴミデータを自動クリーンアップ。Fuzzy Match（類似度85%以上）により、表記ゆれにも対応。
- **フリッカー防止＆言語ガードレール**: `fastText` による判定と、翻訳失敗時の自動リトライ機能を搭載。記号やノイズを遮断し、チラつきのない安定したオーバーレイを実現します。

## 必須環境

- **OS**: Windows 10 または Windows 11 (WinRT OCRの使用に必須)
- **Python**: Python 3.10 以上
- **Ollama**: ローカルにインストールされ、バックグラウンドで起動している必要があります。事前に以下のコマンドでモデルをダウンロードしてください。
  ```bash
  ollama run translategemma:4b
  ```

## インストールと使い方

1. **リポジトリのクローン:**
   ```bash
   git clone <repository_url>
   cd Real_Time_Translate
   ```

2. **依存パッケージのインストール:**
   仮想環境（venv等）の使用を強く推奨します。
   ```bash
   pip install -r requirements.txt
   ```

3. **設定の変更 (任意):**
   `config.json` を編集することで、対象のウィンドウ名、翻訳言語、使用するLLMモデル、UIのフォントサイズなどをカスタマイズできます。
   ```json
   {
     "window_title": "翻訳対象のウィンドウタイトル",
     "target_language": "ja",
     "translation_model": "translategemma:4b"
   }
   ```

4. **アプリの起動:**
   ```bash
   python main.py
   ```
   起動すると、指定したウィンドウに透明なオーバーレイが重なり、自動的にリアルタイム翻訳が開始されます！

## 動作の仕組み

1. **画面キャプチャ**: 指定されたウィンドウの矩形を定期的にキャプチャします。
2. **OCR抽出**: プランに応じて WinRT OCR (2-Pass) や PaddleOCR を使い分け、文脈に応じた最適な行結合を行います。
3. **バリデーション**: `fastText` が「翻訳すべき文章か」を判定し、記号やノイズを排除。翻訳失敗時は自動リトライで品質を確保します。
4. **翻訳キュー**: キャッシュ（Fuzzy match含む）にないテキストのみを Ollama へ送信。起動時リセットにより常に最新の状態を保ちます。
5. **オーバーレイ描画**: バイナリサーチ的なフォントサイズ調整により、元の枠内に最大の文字サイズで描画します。

## OBS Studioでの配信・録画設定

この翻訳オーバーレイをOBS Studioでキャプチャして配信や録画に載せるには、「ブラウザソース」機能を使用します。

1. OBSの「ソース」から **「ブラウザ」** を追加します。
2. 「URL」の欄に `http://localhost:5001/overlay` と入力します。
3. 「幅」と「高さ」を、対象のゲーム画面やモニターの解像度（例: 1920 / 1080）に合わせます。
4. （任意）必要であれば「カスタムCSS」の欄を空にしてください。
5. OBSのプレビュー画面で、ゲーム画面ソースの上にブラウザソースが重なるように配置してください。
