# RTT ロジック・挙動詳細確認メモ (v1.1.4 安定化分析)

本メモは、翻訳表示の「張り付き」および「ゴースト」現象の原因を特定するために、現在のコード（`main.py`, `src/ui.py`）内の主要な判定ロジックをリストアップしたものです。

---

## 1. 表示のライフサイクルと判定基準

### 1.1 IDの生成と固定 (Tracking)
- **ID生成ロジック** (`main.py: 1097行目付近`): 
    - `chunk['id']` は、読み取った文字列のハッシュ（先頭8文字）と、座標を20px単位で丸めた `area_id` を組み合わせて生成されます。
- **ID固定（空間トラッキング）** (`main.py: 1062-1096行目`):
    - 前フレームの `history_chunks` と比較し、`IoU > 0.1` かつ **`類似度 > 0.80`** であれば古いIDを継承します。
    - **[重要]**: 類似度が 0.80 未満の場合は「新しいID」が発行されます。
    - **テンプレート画像の永続化 [v1.1.4 修正]**: 翻訳処理の完了コールバック（`_on_single_translation_done`）実行時、キャッシュ保存の過程で `overlay_state` の該当 ID 項目から追従用画像データ `step3_crop` および `parent_rect` が上書きによって消失（リセット）していた致命的バグを修正。翻訳完了後も追従パラメータを確実に引き継ぐことで、字幕が元の位置に巻き戻ったり見失ったりする問題を根絶しました。

### 1.2 重複排除と表示決定 (Pre-processing)
- **同一フレーム内の重複排除** (`main.py: 1178-1204行目`):
    - OCRの結果（`chunks`）の中で、`IoU > 0.7` 以上重なっている枠があれば、面積が大きい方を優先し、小さい方を破棄します。
    - また、一方が他方に80%以上包含されている場合も、小さい方を破棄します。
- **キャッシュ即時表示** (`main.py: 1121-1140行目`):
    - `translation_cache` にヒットすれば、翻訳を待たずに `overlay.show_translation()` を呼び出します。

### 1.3 翻訳完了後の連鎖表示 (Broadcasting)
- **連鎖表示ロジック** (`main.py: 1895-1919行目`):
    - 翻訳が完了した際、`active_translations`（現在生きている全ID）を走査し、**「原文が一致するすべてのID」**に対して、同一の翻訳結果を表示します。
    - **[ゴースト原因]**: 座標がわずかに違うために重複排除を抜けてしまった「別ID」が存在すると、この瞬間に複数のサイズ違いの翻訳が同時に出現します。

## 2. 消去のロジックと判定基準

### 2.1 空間的な追い出し (Spatial Eviction)
- **重なり判定** (`main.py: 1208-1306行目`):
    - 既存の表示（`overlay_state`）と新しい枠が **`IoA > 0.3`**（30%以上の包含）で重なっている場合に発動します。
- **不一致パージ (2ストライク制)** (`main.py: 1268-1284行目`):
    - 類似度が `0.6` 未満の変化を2回（2フレーム）連続で検知すると、古い表示を削除します。
- **高品質上書き** (`main.py: 1290行目付近`):
    - 新しい枠のOCR品質スコアが旧枠より `0.4` 以上高い場合、ストライクを待たずに古い方を削除します。

### 2.2 不在判定のハイブリッド化 (Existence Window) [v1.1.2 刷新]
- **判定ロジック** (`main.py: 883-936行目付近`):
    - **フェーズ1: テンプレートマッチング追従 (OpenCV)**:
        - `step3_crop` (前回の枠内画像) を使い、現在の画面内で一致箇所（`max_val > 0.75`）を探します。
        - 一致すれば移動量 `(dx, dy)` を算出し、`is_missed = False` を確定します。
    - **フェーズ2: エッジ密度救済 (Edge Density Recovery)**:
        - OpenCV が見失った場合（`is_missed = True`）でも、元の位置でのエッジ密度変化率（`density_diff`）が **25%未満** であれば、「文字は変わらずそこに存在している」とみなして `is_missed = False` に救済します。
        - **[v1.1.2 調整]**: 高解像度化に伴う微細なノイズに対応するため、救済範囲を 10% → 25% へ緩和。
    - **フェーズ3: 強制消去判定**:
        - エッジ変化率が **50%以上** 激変、または密度が **0.005未満**（ほぼ空白）になった場合、OpenCV の結果に関わらず `is_missed = True`（消失確定）とします。
        - **[v1.1.2 調整]**: 誤認識による即時消去を防ぐため、激変の閾値を 30% → 50% へ引き上げ。
    - **フェーズ4: WGCフレーム・絶対密度消失判定 [v1.1.4 導入]**:
        - テンプレートマッチングや通常のエッジ密度変化（相対値）とは別に、WGC(DXCAM)バッファからクロップした現在表示中の字幕エリアを直接取得し、Canny エッジ密度を計算。
        - OCR実行時の基準密度 `base_density` と比較し、**残存率 `retention_ratio` が 15% 未満に急落** した場合、タイムスライディングによる「不在履歴のストライク制（猶予期間）」を完全にバイパスし、**即座に `force_evict = True` にてパージ（即時消去）** を行います。
        - これにより、シーンチェンジやメニュー閉鎖時に「文字が消えたのに古い翻訳が画面に残る（ゴースト現象）」が完全に解消されました。
- **不在履歴のストライク制**:
    - 上記判定による `is_missed` フラグを直近6フレーム保持し、以下の条件で UI から削除します。
    - **条件1: 3回連続不在**（完全な消失）
    - **条件2: 直近6回中4回不在**（激しいチラつき・不安定）
- **長期ゴースト対策・深層再検証 (Deep Verification) [v1.1.4 導入]**:
    - 画面に表示され始めてから **15秒以上** 経過している長寿翻訳枠に対し、`_on_tick` モニターの末尾で PaddleOCR の「検出のみ (`rec=False`)」をバックグラウンド実行します。
    - 検出されたブロック群との IoU 突合において一致する枠が全く存在しない場合、何らかのロジック抜けや応答ラグで残ったゴーストとみなし、即座に `_purge_cid()` を呼び出して該当IDの内部データ、履歴、および UI ラベルを完全に物理破棄します。
- **[改善]**: テンプレートマッチングにより「移動」を正確に分離できたため、エッジ密度判定は「内容の消失」に専念できるようになり、追従と消去の両面で安定性が飛躍的に向上しました。

### 2.3 UI（QLabel）の破棄
- **物理破棄** (`src/ui.py: 116-123行目`):
    - `sync_active_ids()` によって、メインスレッド側で削除されたIDに対応する `QLabel` が `deleteLater()` でメモリから解放されます。

## 3. v1.1.2 強化アップデート（エッジ統合と判定重みの最適化）

### 3.1 エッジハイブリッド型ID管理
- **ID生成の精緻化** (`main.py: 1107-1113行目`):
    - `area_id` に **エッジ密度（25段階、0.04刻み）** を組み込みました。
    - 書式: `{text_hash[:8]}_{gx}_{gy}_{density_step}`
    - これにより、内容や座標が同じでも、背景の複雑さ（視覚的特徴）が異なる場合に別IDとして管理しやすくなりました。
- **ID固定（トラッキング）の判定緩和** (`main.py: 1084-1100行目`):
    - 比較対象: 今回のPaddleOCR時点のエッジ vs 前回のOCR時点のエッジ。
    - **[Same判定(弱め)]**: エッジ誤差が **8%以内** かつ テキスト類似度が **0.7以上** であればIDを継承します。
    - これにより、OCRの微細な読み違い（揺らぎ）によるID跳ねを抑制します。

### 3.2 空間パージ判定の高度化 (Spatial Eviction v2)
- **エッジ優先の同一性判定** (`main.py: 1306-1320行目`):
    - **[Same認定(慎重)]**: `(エッジ誤差 < 5% かつ テキスト類似度 > 0.5)` または `(テキスト類似度 > 0.85)` の場合に同一物とみなします。
    - エッジが似ているだけでは同一視せず、最低限のテキスト一致も求めることで、背景が共通する別文章への誤った「張り付き」を防ぎます。
- **不一致判定の加速** (`main.py: 1343-1357行目`):
    - **[Different判定(標準)]**: テキスト類似度が 0.6 未満、かつ **エッジ誤差が 20% 以上** ある場合は、即座に 2ストライク（即削除確定）を付与します。
- **キャッシュ・ファストパス (即時パージ) [v1.1.2 強化]**:
    - **判定**: 類似度が 0.6 未満であっても、以下のいずれかを満たす場合はストライクを待たずに **即座に mismatch_strikes = 2 を設定してパージをトリガー** します。
        - 読み取った新しいテキストが既にキャッシュに存在する場合（即座に正しい翻訳に差し替えるため）。
        - テキスト類似度が **0.25 未満**（完全に別物のテキスト）である場合。
    - これにより、メニューの切り替え時などに「枠の位置は同じだが中身が全く違う」場合に、古い翻訳が居座る現象を解消します。

### 3.3 情報同期の徹底
- **エッジ取得のタイミング変更** (`src/ocr.py: 254-266行目`):
    - `base_density` の計算を、メインループ（6FPS）から **PaddleOCRの精読フェーズ（2.3）** へ移動しました。
    - PaddleOCRが「実際に解析したその瞬間の画像」からエッジを抽出するため、テキスト内容とエッジ数値の間にタイムラグ（時間的な不一致）がありません。
- **短期ブラックリスト (Zombie Prevention)** (`main.py: 1337-1342行目`):
    - 不一致パージによって消去された原文を **5秒間ブラックリスト** に登録します。
    - 登録期間中は、キャッシュヒットや翻訳完了後のブロードキャストを物理的に遮断し、ゴーストの復活を根絶します。

### 3.9 インテリジェント・パーティショニング (Subdivision v2) [v1.1.2 最適化]
- **分割判定ロジック** (`src/ocr.py: 374-388行目付近`):
    - **フォントサイズ主軸**: 隣接する要素間で文字の高さが **1.5倍** 以上異なる場合を「UIパーツの境界」とみなし、優先的に分割します。
    - **空間判定の緩和**: 以前の 1.0〜2.0倍 という厳しい閾値を **4.0倍** へ大幅に緩和。これにより、通常の改行やフレーズ間のスペースで文章が不自然に細切れになるのを防ぎ、PaddleOCR 本来のまとまり（コンテキスト）を維持します。
- **効果**: 「一続きの文章」は一つの枠として維持しつつ、「タイトルと本文」「ラベルと値」といった視覚的に分離すべき要素だけを正確に切り出せるようになりました。

### 3.4 同一フレーム内重複排除の優先順位刷新
- **優先順位の変更** (`main.py: 1195-1204行目`):
    - **1. 現在表示中の内容との同一性**: 今画面に出ている内容を維持できるものを最優先。
    - **2. 安定性**: すでにIDが発行され表示中のもの。
    - **3. OCR品質スコア**: 記号の少なさ等。
    - **4. サイズ**: 面積。
    - 面積よりも「今出ている翻訳との整合性」を優先することで、サイズ違いの枠による上書き（ゴーストの予備軍）を発生源でカットします。

### 3.5 エッジ優先の事前判定 (Pre-check) [v1.1.2 強化]
- **ロジック** (`main.py: 947-965行目`):
    - OCR（Discovery）を起動するかどうかの判断を、エッジ変化率を主役としたフローに変更しました。
    - **ステップ1 (エッジ激変)**: エッジ変化が **10%以上** なら、背景に関わらず強制実行。
    - **ステップ2 (エッジ安定)**: エッジ変化が **2%未満** なら、背景が動いていても強制スキップ。
    - **ステップ3 (ピクセル安定)**: エッジ変化が中程度の場合のみ、修正版ピクセル差分（L1ノルム）を確認し、静止していればスキップ。
    - これにより、3Dスクロール等の背景変化に惑わされず、文字の変化のみを捉えてOCRをサボることが可能になりました。

### 3.6 テキスト正規化とキャッシュ一致の拡張 [v1.1.2 強化]
- **正規化の強化** (`main.py: 81-96行目`):
    - OCRノイズとなりやすい記号（`. , : ; ! ? ' " ` ^ ~` など）を正規化対象に追加。
- **全長のキャッシュ一致解禁** (`main.py: 1532行目`):
    - これまで長文（20文字以上）に限定していた正規化一致を、**すべてのテキスト長**に解禁しました。
    - これにより、UIボタン等の短文で末尾にドットがつくようなOCRの揺れが発生しても、キャッシュにヒットさせて再翻訳（と表示のチラつき）を阻止します。

## 4. 情報の保存状態 (State Management)

| データ名 | 保存場所 | 役割 | パージ時の挙動 |
| :--- | :--- | :--- | :--- |
| `translation_cache` | `main.py` | `原文: 翻訳文` の辞書 | **永続。消去されない。** |
| `overlay_state` | `main.py` | 表示中の詳細（座標、親枠、追従画像、ストライク等） | 明示的に `del` される。 |
| `active_translations`| `main.py` | 生きているIDのリスト | 明示的に `del` される。 |
| `active_labels` | `src/ui.py` | IDとQLabelの対応 | `deleteLater()` で破棄される。 |
| `step3_crop` | `overlay_state` | 追従用のグレースケール画像 (numpy.ndarray) | ID削除時に一緒に消える。 |


### 3.7 OCR抽出フェーズの厳格な分離 (Engine Responsibility) [v1.1.2 強化]
- **WinRT OCR (Discovery Only)**: 索敵フェーズにおいて枠を検出するためだけに使用。WinRTが読み取った「テキスト」は即座に破棄されます。
- **PaddleOCR (Recognition Only)**: WinRTが提示した全ての枠に対して、PaddleOCRが再度精読を行います。
- **目的**: 翻訳のソースとなるテキストを PaddleOCR 1本に絞ることで、エンジンの癖による表記揺れや重複発生を根絶します。

### 3.8 翻訳前重複排除 (Strict Deduplication) [v1.1.2 導入]
- **実装場所**: `_poll_ocr_result` (main.py)
- **多層フィルタ**:
    - `seen_in_this_batch`: 今回のOCRで見つかった同じテキストを統合。
    - `self.pending_texts`: 現在翻訳中のテキストとの重複をブロック。
    - `self._recent_texts` (TTL: 2s): 直近2秒間に翻訳キューへ送ったテキストをブロック。
- **効果**: シーンチェンジ時に古いテキストが何度も再送され、ゴーストがループする現象を完全に抑止。
